# 问题描述

执行 `python main.py init`（或等价初始化流程）后，`vadmin_auth_user` 中管理员等用户的 `password` 字段仍为**明文**，未写入 bcrypt 哈希。

# 根因分析

1. **Excel 单元格类型**：密码列若为数字（如纯数字密码），原逻辑要求 `isinstance(pw, str)` 才哈希，导致**整型/浮点原样进入插入流程**，库中表现为可读明文或错误类型。
2. **表头别名**：表头为「密码」等中文或非严格 `password` 时，原逻辑只读 `r.get("password")`，**未合并别名列**，哈希分支未执行。
3. **工作表名**：`init.xlsx` 中 sheet 名与逻辑表名 `vadmin_auth_user` 不完全一致时，`datas` 取错或为空，可能与其他导入方式混用，表现为「初始化后仍是明文」。

# 涉及文件

- `kinit-api/scripts/initialize/user_password.py`：合并密码列别名、类型规范化、bcrypt 判定与哈希。
- `kinit-api/scripts/initialize/initialize.py`：`_rows_for_table` 按表名匹配 sheet。
- `kinit-api/test/test_core/test_login_identifier.py`：补充整型、`密码` 表头等用例。

# 设计约束更新

- 初始化设计补充：`init.xlsx` 用户表密码列支持表头别名 `password` / `密码` 等；数字单元格一律按字符串规则哈希（已为 `$2a$`/`$2b$`/`$2y$` 前缀的 bcrypt 串不重复哈希）。
