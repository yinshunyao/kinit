# 功能设计

## 登录标识（密码登录）

- 请求体字段名**不变**：仍为 `telephone`（与历史客户端兼容），语义为「登录标识」，可为**国内手机号**或**账号**。
- **判定规则**：
  - 若标识经校验为**国内 11 位手机号**（与现有 `vali_telephone` 规则一致），则按 `vadmin_auth_user.telephone` **精确匹配**查询用户。
  - 否则，若标识中**至少包含一个 ASCII 字母**（`[a-zA-Z]`），则视为**账号登录**：先按 `name` **精确匹配**，若无记录再按 `nickname` **精确匹配**（兼容历史种子数据将展示用账号写在「昵称」、姓名写在 `name` 的情况，如 `name=kinit`、`nickname=admin`）。
  - 其余情况（如 11 位全数字但不符合号段规则、且无字母）：拒绝并提示格式错误（如「请输入正确手机号」或「账号须包含字母」），不查库。
- **短信登录**（`method=1`）：标识**必须**为合法国内手机号，仅按 `telephone` 查询；逻辑与改造前一致。

## 错误提示

- 密码登录：用户不存在与密码错误统一为 **「账号或密码错误」**，避免区分枚举用户。
- 短信登录：用户不存在仍为 **「该手机号不存在！」**；验证码错误仍为 **「验证码错误」**。
- 账号冻结、无工作人员权限：文案统一为 **「此账号已被冻结」**、**「此账号无权限」**（不再强调「手机号」）。

## OAuth2 `/api/login`

- `username` 参数承载与 `/login` 密码登录相同的登录标识，判定与查库规则同上。

## 初始化脚本 `scripts/initialize`

- 读取 `init.xlsx` 中 `vadmin_auth_user` 工作表数据时，由 `scripts/initialize/user_password.py` 的 `prepare_vadmin_auth_user_rows` 预处理：
  - **表头**：除 `password` 外，支持常见别名（如 `密码`、`Password` 等），合并为 ORM 字段 `password` 再处理。
  - **单元格类型**：数字（整型/浮点，如 Excel 将密码存为数值）会先规范为字符串再判断是否哈希；**非 bcrypt 明文**一律 `passlib` bcrypt；已符合 `$2a$`/`$2b$`/`$2y$` 且长度足够的串视为已有哈希，**不重复哈希**。
- **工作表名**：`InitializeData` 按表名取数时，除完全匹配外，支持 sheet 名与表名在「去下划线、横杠、空格后小写相同」时的匹配（例如避免因命名细微差异未命中导致未走预处理）。

# 数据模型

- 无表结构变更；使用既有 `telephone`、`name`、`nickname`、`password` 字段。

# 约束与边界

- `name` / `nickname` 非全局唯一时，`get_data` 仅返回第一条匹配记录；需在运维上避免多条记录共用同一登录账号字符串。
- 纯中文姓名无英文字母时不能作为账号登录标识，须使用手机号登录。

# 前端（kinit-admin / kinit-uni）

- 密码登录页：去掉默认手机号与密码；占位与文案体现「手机号或账号」；适当放宽标识输入长度上限（如 50）。
- 短信登录页：仍为手机号专用。
